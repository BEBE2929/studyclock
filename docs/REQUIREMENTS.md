# スタディクロック 要件設計定義書

**バージョン**: 1.0  
**作成日**: 2026-06-07  
**対象**: 小学1年生向け アナログ時計学習Webアプリ

---

## 1. プロジェクト概要

### 1.1 目的

小学1年生（6〜7歳）がアナログ時計の読み方を、段階的かつ楽しく習得できるWebアプリを提供する。
KUMON式学習時計にならい、時計の文字盤デザイン自体が難易度に応じて変化することで、補助なしで時計を読む力を自然に育てる。

### 1.2 対象ユーザー

| ユーザー | 利用シーン |
|---------|-----------|
| 小学1年生（6〜7歳） | メインの学習者。ひとりで操作できるシンプルなUIが必要 |
| 保護者 | 子どもの進捗確認、学習サポート |
| 教師 | 授業補助ツールとしての利用 |

### 1.3 学習目標

1. 短い針（時針）が何時を指しているかわかる
2. 「ちょうど」「はん（30分）」の時刻が読める
3. 5分刻みの時刻が読める
4. 文字盤の補助なしで任意の時刻が読める

## 2. 機能要件

### 2.1 学習モード（じかんをおぼえよう）

| 項目 | 仕様 |
|------|------|
| 表示内容 | アナログ時計 + 時刻テキスト（ひらがな）を同時表示 |
| 操作 | 「つぎへ」ボタンで別の時刻に切り替え |
| 出題範囲 | 選択中のレベルに対応した時刻のみ表示 |
| 解説表示 | 時針・分針をそれぞれ色で区別し、役割をテキストで説明 |
| 針のアニメーション | 時刻が切り替わるとき、針がなめらかに動く |

### 2.2 クイズモード（もんだいをとこう）

#### 読み取りクイズ（よんでみよう）
- 時計を表示し、正しい時刻を4択の中から選ぶ
- 選択肢はひらがな表記（例:「1じ30ぷん」）
- 正解: 正解アニメーション + 効果音（オプション）+ 次の問題へ
- 不正解: 不正解アニメーション + 正解を表示してから次の問題へ

#### セット針クイズ（あわせてみよう）
- 時刻テキストを表示し、針をドラッグして指定時刻に合わせる
- 許容誤差: Level 1〜2 は±5分、Level 3 は±2分、Level 4 は±1分

#### クイズ共通仕様
- 1セット10問
- 正解数でスコアを計算し、星評価を決定
- 途中でやめても進捗は記録しない（セット完了時のみ保存）

### 2.3 難易度レベル

難易度に応じて **時計の文字盤デザインが変わる**。補助情報が段階的に減り、最終的には通常のアナログ時計と同じ状態になる。

#### レベル1（にゅうもん）— 入門
| 項目 | 仕様 |
|------|------|
| 文字盤 | 内側: 時の数字（1〜12、赤・大きめ）<br>外周: 分の数字（0〜59、青・小さめ）<br>5分ごとにカラーセクションでハイライト（黄・青交互） |
| 出題範囲 | ちょうどの時刻のみ（1じ、2じ…12じ） |
| 解除条件 | 最初から利用可能 |

#### レベル2（しょきゅう）— 初級
| 項目 | 仕様 |
|------|------|
| 文字盤 | レベル1と同じ（全分数字 + カラーハイライト） |
| 出題範囲 | ちょうど + 30分（1じちょうど、1じはん） |
| 解除条件 | レベル1で★1以上を獲得 |

#### レベル3（ちゅうきゅう）— 中級
| 項目 | 仕様 |
|------|------|
| 文字盤 | 外周に5分刻みの数字のみ（0, 5, 10, 15…55）。1分単位の数字は非表示 |
| 出題範囲 | 5分刻みの全時刻（1じ5ふん、1じ10ぷん…） |
| 解除条件 | レベル2で★1以上を獲得 |

#### レベル4（じょうきゅう）— 上級
| 項目 | 仕様 |
|------|------|
| 文字盤 | 分の数字なし。目盛りと時の数字のみ（通常のアナログ時計と同じ） |
| 出題範囲 | 1分刻みの全時刻（任意の時刻） |
| 解除条件 | レベル3で★1以上を獲得 |

#### 文字盤のデザイン仕様（レベル1〜2）

```
        [外周: 分数字リング 0〜59 青色]
           ┌─────────────────┐
           │  [カラーセクション]  │  ← 5分ごとに黄・青交互
           │   [時数字 1〜12]   │  ← 赤・大きめ
           │       [針]        │
           └─────────────────┘
```

- 外周分数字: 0（12時位置）から時計回りに1〜59を配置
- カラーセクション: 0〜4、5〜9、10〜14… の5分ブロックを交互に色分け
- 時の数字: 内側リングに1〜12（赤）

### 2.4 進捗・スコア管理

#### スコア計算
| 正解数 | 星評価 |
|--------|--------|
| 9〜10問 | ★★★ |
| 7〜8問 | ★★ |
| 5〜6問 | ★ |
| 0〜4問 | 星なし |

#### データ永続化
- `localStorage` にJSONで保存
- キー: `studyclock_progress`
- ページリロード・ブラウザ再起動後も保持

#### 進捗の表示
- ホーム画面の各レベルカードに現在の星評価を表示
- ロック中のレベルは鍵アイコンで表示

## 3. 非機能要件

### 3.1 対応環境
| 項目 | 要件 |
|------|------|
| 対応ブラウザ | Chrome / Safari / Edge の最新版 |
| 対応デバイス | PC（1024px以上）、タブレット（768px〜） |
| ネットワーク | 初回読み込み後はオフラインでも動作可能（PWA対応は任意） |

### 3.2 UI・UX要件
- **文字**: ひらがな中心（小学1年生が読める平易な表現）
- **ボタン**: 最小タップ領域 44×44px 以上
- **配色**: 明るく子供が親しみやすいカラーパレット
- **フォント**: Noto Sans JP（視認性が高い日本語フォント）
- **フィードバック**: ボタン押下時に視覚的フィードバック（色変化・スケール変化）

### 3.3 パフォーマンス
- 初回表示: 3秒以内（一般的な回線環境）
- 時計アニメーション: 60fps を目標

### 3.4 アクセシビリティ
- 色だけに依存しない情報伝達（形状・テキストも併用）
- alt テキストの設定

## 5. 技術スタック

| カテゴリ | 採用技術 | 選定理由 |
|---------|---------|---------|
| フレームワーク | Next.js 15（App Router） | SSRとCSR双方に対応、豊富なエコシステム |
| 言語 | TypeScript | 型安全性による品質向上 |
| スタイリング | Tailwind CSS | ユーティリティクラスで高速UI構築 |
| 時計描画 | SVG（Reactコンポーネント） | 解像度非依存、アニメーションしやすい |
| アニメーション | CSS Transitions + Framer Motion | 針の動きを滑らかに表現 |
| 状態管理 | React useState / useContext | 小規模アプリに適切、外部ライブラリ不要 |
| データ永続化 | localStorage | サーバー不要でシンプルに実現 |
| フォント | Noto Sans JP（Google Fonts） | 日本語の視認性が高い |
| パッケージ管理 | npm |  |

## 6. データモデル

```typescript
// 進捗データ（localStorageに保存）
type LevelProgress = {
  level: 1 | 2 | 3 | 4;
  stars: 0 | 1 | 2 | 3;       // 0: 未クリア
  bestScore: number;            // 最高正解数（0〜10）
  attempts: number;             // 挑戦回数
  unlocked: boolean;            // 解除済みかどうか
};

type AppProgress = {
  levels: Record<1 | 2 | 3 | 4, LevelProgress>;
  lastUpdated: string;          // ISO8601
};

// クイズ問題
type QuizQuestion = {
  hour: number;    // 1〜12
  minute: number;  // 0〜59
  type: 'read' | 'set';
};

// 時計表示設定（レベルによって変わる）
type ClockDisplayConfig = {
  showAllMinuteNumbers: boolean;   // 0〜59の全分数字を表示
  showFiveMinuteNumbers: boolean;  // 5分刻みの数字を表示
  showColorSections: boolean;      // カラーセクションを表示
};

const CLOCK_CONFIG: Record<1 | 2 | 3 | 4, ClockDisplayConfig> = {
  1: { showAllMinuteNumbers: true,  showFiveMinuteNumbers: true,  showColorSections: true  },
  2: { showAllMinuteNumbers: true,  showFiveMinuteNumbers: true,  showColorSections: true  },
  3: { showAllMinuteNumbers: false, showFiveMinuteNumbers: true,  showColorSections: false },
  4: { showAllMinuteNumbers: false, showFiveMinuteNumbers: false, showColorSections: false },
};
```

## 9. 検証方法

1. `npm run dev` でNext.jsを起動し、ブラウザで動作確認
2. 各レベルの文字盤が正しく切り替わること
3. クイズの問題が選択レベルの出題範囲内であること
4. `localStorage` に進捗が保存され、リロード後も維持されること
5. レベルのロック解除条件が正しく機能すること
6. Chrome DevTools でタブレット幅（768px）のレスポンシブ表示を確認
